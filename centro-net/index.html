Introducción

El propósito de este ejercicio es diseñar una página web que reúna y muestre todos tus enlaces sociales en un solo lugar.

Si no utilizas redes sociales o prefieres mantener tu privacidad, puedes listar enlaces a páginas de tu artista favorito, empresa, servicio u otro recurso de interés. Considera incluir:

  1. Al menos tres enlaces a redes sociales (Instagram, X, LinkedIn, etc.).
  2. Un enlace de contacto vía correo electrónico.
  3. Un enlace de contacto vía WhatsApp.
  4. Un enlace de contacto vía teléfono.
  5. Una foto de perfil, logo o avatar.
  6. Una imagen de background.
  7. Cualquier uso de recursos adicionales serán considerados.

💀 Deadline: 25 de marzo del 2025

Estructura y Semántica

  1. Usar HTML semántico con etiquetas adecuadas (<header>, <main>, <section>, <footer>, etc.).
  2. Utilizar atributos obligatorios como title en enlaces y alt en imágenes para mejorar la accesibilidad.
  3. Optimizar la jerarquía de títulos.
  4. Organizar tus archivos en carpetas y nombrarlos adecuadamente:
    • Usar minúsculas.
    • Evitar espacios; utiliza guiones medios (-) o guiones bajos (_).
    • Ser descriptivo y conciso.
    • Emplear extensiones de archivo soportadas por navegadores (consulta caniuse.com).

Metadata

  • Incluye un título de página.
  • Incluye definición de lenguaje.
  • Incluir un favicon compatible con navegadores modernos y un fallback para versiones antiguas. Puedes ayudarte de generadores como realfavicongenerator.net
  • Agregar metadata Open Graph para mejorar la previsualización en redes sociales.

Mobile First

  • Priorizar la visualización en dispositivos móviles.
  • Utilizar media queries y la etiqueta picture para imágenes adaptativas.
  • Definir tamaños adecuados para enlaces y texto que faciliten la interacción táctil.

Accesibilidad y Diseño

  • Elegir tipografías y tamaños adecuados para mejorar la legibilidad. Consulta Font Pair o Google Fonts.
  • Asegurar un contraste adecuado en los colores para facilitar la lectura en modos claro y oscuro.
  • Proporcionar descripciones alternativas en imágenes mediante el atributo alt.
  • Utilizar enlaces descriptivos en lugar de textos genéricos como "haz clic aquí".

Multimedia

Prepara y optimiza tus archivos de imagen y video para la web:

  • Utilizar imágenes en formatos modernos (WebP, AVIF) con fallback para compatibilidad.
  • Implementar imágenes responsivas adaptadas a diferentes resoluciones.
  • Redimensiona los archivos a tamaños adecuados para cada dispositivo.
  • Comprime los archivos para reducir el tiempo de carga.

Repositorio en GitHub

El código de este proyecto debe estar alojado en un repositorio público en GitHub para su revisión y consulta. Considera:

  • El repositorio debe ser público.
  • Incluir un archivo README.md con una descripción y enlaces relevantes.
  • Realizar commits descriptivos que reflejen los cambios (evitar mensajes genéricos como "update" o "fix").
  • Publicar el proyecto en GitHub Pages para su visualización en línea.
  • Incluir un enlace al sitio en la descripción del repositorio.

Tips

¿No sabes por dónde empezar? Aquí tienes algunas sugerencias:

  • Crea un moodboard para definir el estilo de tu página.
  • Utiliza herramientas como Adobe Color o Coolors para encontrar la paleta adecuada.
  • Busca inspiración en servicios como Linktree, Heylink o Beacons.ai.
  • Puedes agregar un toque personal integrando un embed de tu canción o video favorito.

Recomendaciones

Considera las siguientes recomendaciones para mejorar la experiencia de desarrollo y la usabilidad del sitio:

  • Evita enlaces rotos o que requieran autenticación para ser vistos.
  • Personaliza los enlaces con estilos visuales distintivos.
  • Incluye una breve descripción "Sobre mí" o añade un archivo humans.txt para brindar mayor contexto.
